思考题

Thinking 5.1 如果通过 kseg0 读写设备，那么对于设备的写入会缓存到 Cache 中。这是一种错误的行为，在实际编写代码的时候这么做会引发不可预知的问题。请思考：这么做这会引发什么问题？对于不同种类的设备（如我们提到的串口设备和 IDE 磁盘）的操作会有差异吗？可以从缓存的性质和缓存更新的策略来考虑。

当我们想操作硬件，即对硬件/外设进行读写时，如果还使用缓存会导致数据在写入时被暂时保存在缓存中，而不是立即写入设备。这种延迟对于要求实时性的设备（如串口）是不可接受的，因为设备期望数据能够立即被处理。因此我们把硬件的物理地址转换为kseg1内核地址，使得CPU得知这部分读写需立即操作而不要用缓存。

串口设备读写频繁，而IDE磁盘读写频率相对较小，因此串口设备发生错误的概率要远大于IDE磁盘。

Thinking 5.2 查找代码中的相关定义，试回答一个磁盘块中最多能存储多少个文件控制块？一个目录下最多能有多少个文件？我们的文件系统支持的单个文件最大为多大？

文件最大大小：共BLOCK_SIZE_BIT / sizeof(uint32_t)个磁盘块用于存储，每个磁盘块又有BLOCK_SIZE_BIT的空间，所以总共就有 BLOCK_SIZE_BIT^2 / 4 的空间可以用于单一文件的存储。

Thinking 5.3 请思考，在满足磁盘块缓存的设计的前提下，我们实验使用的内核支持的最大磁盘大小是多少？

IDE磁盘结构

磁盘块是对磁盘空间的逻辑划分；扇区是对磁盘空间的物理划分

扇区（sector）：磁盘执行读写操作的基本单位，一般是512B。

磁盘块（Block）：虚拟概念，操作系统与磁盘块交互的最小单位；磁盘块大小由操作系统决定，4KB

struct Block {
	uint8_t data[BLOCK_SIZE];//存储一个磁盘块的数据
    //data 数组可以容纳 4096 个字节的数据,按字节存储
	uint32_t type;
} disk[NBLOCK];

IDE磁盘操作

PIIX4 磁盘控制器：基地址为0x180001F0

通过读写 PIIX4 的特定寄存器，我们可以实现以扇区为最小单元的读写。

采用逻辑块寻址（LBA）的方式来进行扇区寻址。 IDE设备将磁盘看作线性的字节序列，每个扇区有其唯一编号。扇区编号有 28 位，因此最多可以寻址 2^28 个扇区，128GB

struct Open {
	struct File *o_file; // mapped descriptor for open file
	u_int o_fileid;	     // file id
	int o_mode;	     // open mode

};

Filefd 是如下所示的结构体。我们在将文件描述符共享到用户进程时，实际上共享的是 Filefd

struct Filefd {
	struct Fd f_fd;

};
serve_init函数

在 serve_init 中，我们就是对 opentab 进行初始化。为其中每个元素设定 o_fileid 和 o_ff。

void serve_init(void) {
	int i;
	u_int va;

	// Set virtual address to map.
	va = FILEVA;

	// Initial array opentab.
	for (i = 0; i < MAXOPEN; i++) {
		opentab[i].o_fileid = i;
		opentab[i].o_ff = (struct Filefd *)va;//只给出了Filefd的虚拟地址，还没有分配物理页
		va += BY2PG;
	}
}

地址的分配是从 va = FILEVA 开始的，其中 #define FILEVA 0x60000000。每个 Filefd 分配一页 BY2PG 的大小。因此所有的 Filefd 存储在 [FILEVA, FILEVA + PDMAP) 的地址空间中。

在 open 中调用 fsipc_open。该函数会将 path 对应的文件以 mode 的方式打开，将该文件的文件描述符共享到 fd 指针对应的地址处。

int read_block(u_int blockno, void **blk, u_int *isnew) {
	//oimt check....
    
    //获取磁盘块编号应该存储到的地址
	void *va = disk_addr(blockno);//我们将 [DISKMAP, DISKMAP+DISKMAX) 的地址空间用作缓冲区，当磁盘读入内存时，用来映射相关的页

	if (block_is_mapped(blockno)) { //编号对应的磁盘块已被映射
		if (isnew) {
			*isnew = 0;
		}
	} else { //如果没有，则我们需要为其分配内存，并将硬盘中的数据读入该内存空间中。
		if (isnew) {
			*isnew = 1;
		}
		try(syscall_mem_alloc(0, va, PTE_D));
		ide_read(0, blockno * SECT2BLK, va, SECT2BLK); // 【磁盘驱动】将硬盘中的数据读入该内存空间
	}

	// Step 5: if blk != NULL, assign 'va' to '*blk'.
	if (blk) {
		*blk = va;
	}
	return 0;
}
