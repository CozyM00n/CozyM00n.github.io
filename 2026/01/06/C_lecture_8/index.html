限定符在函数参数中的作用

int const *p 限定的是 *p，从而向调用者保证这个函数不会修改传进来的数据。

restrict 限定符

T* restrict O;

restrict限定符只能用于修饰指针类型。

storage class specifier 存储类型说明符

C语言提供了一系列Storage Specifier 来规定对象可以存在内存的什么区域。

Storage Class Specifier 确定它们声明的名称（Identifier / 标识符）的两个独立属性：storage duration and linkage. 例如：

  • static - static duration and internal linkage (unless at block scope)
  • extern - static duration and external linkage (unless already declared internal)

一些概念：Identifier / 标识符, Scope / 作用域, Name Space / 命名空间

Identifier / 标识符

标识符用于定位(designates )一个或多个实体(entities)。

8种实体 / entity :

Object, Function, Label Name, Tag , Member of Structure Typedef Name ,Macro Name ,Macro Parameter

标记（tag）：定义struct, union或enum关键字之后使用的标识符。

Scope / 作用域

For each different entity that an identifier designates, the identifier is visible (i.e., can be used) only within a region of program text called its scope.

对于由标识符标识的实体，该标识符仅在其作用域（Scope）可见 (i.e. 可使用)。

C语言共 4 种 scope：

1.Function Scope
  • A label name is the only kind of identifier that has Function Scope.
  • 在C语言中，标签（label）主要用于两个目的：goto语句的目标和 switch 语句的case或default子句
  • 只在函数内部有效
2.Function Prototype Scope / 函数原型作用域

函数原型(函数声明)中的参数名具有 function prototype scope，只用于类型检查，在 ) 处结束。

3.Block Scope

Inside a block({}) or within the list of parameter declarations

4.File Scope

不在任何Block（大括号{}）中 && 不在任何函数参数表中

Name Space / 命名空间

同样的 identifier 可以属于不同的 name space，包括以下Name Space：

  • label name （goto）
  • tag（ struct、union和enum的名称）
  • member of structures or unions结构体/联合体的成员
  • ordinary identifiers (eg.变量名，函数名，函数参数，宏定义，typedef类型名，enum成员等)

Linkage

相同的标识符出现在不同的地方，它们是否表示同样的实体？——通过Linkage分辨

Linkage refers to the ability of an identifier (variable or function) to be referred to in other scopes.

If a variable or function with the same identifier is declared in several scopes, but cannot be referred to from all of them, then several instances of the variable are generated. The following linkages are recognized:

链接是指标识符（变量或函数）在其他Scope内被引用的能力。如果在多个作用域中声明具有相同标识符的变量或函数，但无法从所有作用域中引用该变量或函数，则会生成该变量的多个实例。可以识别以下链接：

  • no linkage. The identifier can be referred to only from the scope it is in. All function parameters and all non-extern block-scope variables (including the ones declared static) have this linkage.

    标识符只能从其所在的作用域引用。所有函数参数和所有非外部 块作用域变量都具有此链接。

  • internal linkage. The identifier can be referred to from all scopes in the current translation unit. All static file-scope identifiers (both functions and variables) have this linkage.

    该标识符可以从当前翻译单元 (如一个.c文件) 中的所有范围引用。所有 static 文件范围标识符（函数和变量）都具有此链接。

  • external linkage. The identifier can be referred to from any other translation units in the entire program. All non-static functions, all extern variables (unless earlier declared static), and all file-scope non-static variables have this linkage.

    该标识符可以从整个程序中的任何其他翻译单元引用。所有非静态函数、所有外部变量（除非声明为static）、以及所有file-scope non-static变量都具有此链接。

在不同的作用域或在同一作用域中多次声明的标识符可以通过一种称为“链接”的过程指向相同的对象或函数。

内部链接 internal linkage

条件：

如果一个对象标识符拥有file scope，且被static或constexpr修饰， 则该标识符internal linkage

特点：

如果一个标识符具有Internal Linkage，则其他编译单元无法使用这个标识符，有助于程序的模块化

no linkage

The following identifiers have no linkage:

1、an identifier declared to be anything other than an object or a function;

2、an identifier declared to be a function parameter;

3、a block scope identifier for an object declared without the storage-class specifier extern

1、标识符既不是一个对象又不是函数

2、标识符被声明为函数参数

3、拥有Block Scope且没用extern修饰的标识符

no linkage的标识符都指向不同的对象

extern修饰对象标识符

【易错】

对于一个用extern修饰的标识符，如果该标识符在可见范围内已经指定了内部/外部链接，那么在后续声明中该标识符的链接方式与先前声明中指定的链接方式相同。

extern修饰函数标识符

在Block作用域声明函数标识符必须用extern.

external linkage

全局变量

If no prior declaration is visible, or if the prior declaration specifies no linkage, then the identifier has external linkage.

如果之前没见过这个identifier，或者之前的声明是none linkage，那么这个extern修饰的标识符就是external linkage。
